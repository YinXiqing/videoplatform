# 视频平台部署说明

## 📦 项目位置
项目已放在当前工作目录下：`./`

## 🚀 快速启动

### 方法一：使用 Linux 启动脚本（推荐）
1. 运行 `./start.sh`
2. 按照提示选择操作：
   - 选项1：首次运行，初始化数据库
   - 选项4：同时启动前后端服务器

### 方法二：手动启动

#### 1. 初始化数据库（仅需首次运行）
```bash
cd ./
uv venv backend/.venv
source backend/.venv/bin/activate
cd backend
uv pip install -r requirements.txt
python init_db.py
```

#### 2. 启动后端服务器
```bash
cd ./
source backend/.venv/bin/activate
cd backend
python run.py
```
后端将运行在：http://localhost:5000

#### 3. 启动前端服务器
```bash
cd ./frontend
pnpm install
pnpm start
```
前端将运行在：http://localhost:3000 或 http://localhost:3001

## 🔧 环境要求

### 后端
- Python 3.12+
- uv 包管理器

### 前端
- Node.js 24.x
- pnpm 包管理器

## 📱 访问地址
- **前端界面**：http://localhost:3000 或 http://localhost:3001
- **后端API**：http://localhost:5000
- **管理员后台**：登录后点击"后台管理"

## ⚠️ 注意事项

1. **端口冲突**：如果3000端口被占用，前端会自动使用3001端口
2. **文件上传**：默认最大文件大小 500MB
3. **视频审核**：用户上传的视频需要管理员审核后才能在前端显示
4. **数据库**：使用 SQLite，数据存储在 `backend/database/videoplatform.db`

## 🛠 故障排除

### 问题1：后端启动失败
- 检查 Python 版本是否为 3.12+
- 确保虚拟环境已激活：`source .venv/bin/activate`
- 重新安装依赖：`uv pip install -r requirements.txt`

### 问题2：前端启动失败
- 检查 Node.js 是否安装
- 删除 node_modules 文件夹后重新运行 `pnpm install`
- 确保没有其他程序占用3000或3001端口

### 问题3：无法连接后端
- 确保后端服务器正在运行
- 检查防火墙设置
- 确认后端运行在 http://localhost:5000
